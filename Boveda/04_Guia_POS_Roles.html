🏪 Guía POS, Roles y Seguridad

Fiore de Enero — Sistema de Punto de Venta y Control de Acceso
Treze Labs
Laboratorio de Sistemas Digitales · Abril 2026

Este documento describe la operación del Punto de Venta (POS), el sistema de Control de Acceso por Roles (RBAC) y las políticas de seguridad implementadas para proteger la caja del restaurante.

🔐 1. Sistema de Acceso por Roles (RBAC)

¿Qué es RBAC?

El sistema requiere un PIN numérico para acceder al Command Center. Dependiendo del PIN ingresado, el sistema otorga diferentes niveles de acceso:

RolPINAcceso
Administrador (Dueña) 2026 Acceso total: Dashboard de Ventas, Historial, POS, Mesas Activas, Gestión de Meseros
Cajera 1313 Acceso restringido: Solo POS y Mesas Activas. No ve cifras de ventas ni gestiona personal.
🔒 Los PIN se verifican mediante hash criptográfico SHA-256. Nunca se almacenan en texto plano. Si alguien inspecciona el código, solo verá el hash cifrado, no el PIN real.

💳 2. Operación del POS (Punto de Venta)

Flujo de Cobro

  1. Ingresa al Command Center con tu PIN (Admin o Cajera).
  2. Ve a la sección "Mesas Activas" para ver las mesas que los meseros han abierto.
  3. Haz clic en la mesa que deseas cobrar.
  4. Revisa el detalle del ticket: platillos, cantidades, extras y total.
  5. Selecciona el método de pago (Efectivo, Tarjeta, Transferencia).
  6. Si aplica, ingresa el porcentaje de propina.
  7. Pulsa "Cobrar". La mesa se cierra y la venta queda registrada en la base de datos.

🛡️ 3. Restricciones de la Cajera

Cuando la cajera inicia sesión con su PIN, el sistema oculta automáticamente los siguientes módulos:

  • 📊 Dashboard de Ventas — La cajera no debe ver el resumen financiero diario.
  • 📋 Historial de Cierre — No puede consultar ventas pasadas ni exportar datos.
  • 👥 Gestión de Meseros — No puede crear, editar ni eliminar meseros.

Acciones que Requieren Autorización Gerencial

Las siguientes acciones están bloqueadas para la Cajera. Al intentarlas, el sistema solicita el PIN del Administrador (2026):

  • ❌ Cancelar una mesa completa
  • 🗑️ Eliminar un producto individual del ticket
  • ✏️ Modificar cantidades o descuentos de una mesa activa
Protocolo Operativo: Si un cliente cancela un platillo, la cajera debe solicitar a la dueña que ingrese su PIN de autorización en la misma pantalla. Si la dueña no está presente, el mesero debe avisar directamente a cocina para detener la preparación.

⚠️ 5. Protocolos de Emergencia

Si la cajera se equivocó de mesa al cobrar:

La venta ya quedó registrada en la base de datos. Para revertirla, la dueña debe contactar a Treze Labs para una intervención manual en la hoja de cálculo.

Si se pierde conexión a internet:

El sistema mostrará alertas visuales inmediatas. Ninguna operación se perderá silenciosamente. Si el operador ve un error de red, debe esperar a que se restaure y volver a intentar.

Si un mesero intenta guardar en una mesa ya cobrada:

El sistema detectará la condición de carrera y mostrará un mensaje: "La mesa fue cobrada en caja hace unos instantes." La operación se aborta y no se duplican datos.
